## Context

本项目是一个纯前端雅思拼写训练工具，零后端依赖。用户打开一个静态 HTML 页面即可使用。核心交互是：展示中文释义 → 15 秒倒计时拼写 → 实时检测 → 超时/错误时红色展示正确单词 → 一轮 15 词结束后统计并支持错词复习。

项目作为 OpenSpec 新变更提交，产出物为 index.html + style.css + app.js + data/words.json。

## Goals / Non-Goals

**Goals:**
- 提供一个完整的、可离线运行（除发音外）的单页拼写训练器
- 核心逻辑（中文展示、计时、实时检测、错误处理）完全不依赖外部服务
- 发音通过 dictionaryapi.dev 免费 API 增强，API 不可用时静默降级
- 全部数据持久化通过 localStorage，无需用户登录

**Non-Goals:**
- 不做用户系统、不做后端
- 不做社交/分享功能
- 不做复杂的动画/动效（聚焦功能）
- 不做移动端原生 App（但需要适配移动端浏览器）

## Decisions

### Decision: 纯函数式游戏状态机
采用有限状态机管理整个游戏流程，而非分散的事件监听。状态包括：
```
IDLE → PLAYING → CORRECT | TIMEOUT | WRONG → PLAYING | ROUND_END
```
**原因**: 拼写游戏的状态转换清晰且有限，状态机可以避免竞态条件（如倒计时与输入检测的时序冲突）。

### Decision: 单词级实时检测，非字母级
每次 input 事件触发时，将当前输入与目标单词比较：
- 输入是目标单词的前缀 → 正常状态
- 输入不是目标单词的前缀 → 整框变红
- 输入完全匹配 → 自动通过

**原因**: 用户明确要求"整个词不匹配才标红"，而非逐字母反馈。这种策略实现简单且符合学习心理——用户知道错了但可以继续敲完看到差距。

### Decision: 倒计时使用 requestAnimationFrame / setInterval 而非 Date.now 差值
**原因**: 15 秒总时长较短，setInterval 100ms 精度已足够，且实现更直观。同时使用时间戳差值做校正避免 drift。

### Decision: API 调用延迟加载
每个词展示后才请求 dictionaryapi.dev，而非启动时批量请求全部 500 词的发音。
**原因**: 节省带宽、避免被限流、大部分词在单次会话中可能不会被用到。每次请求结果做简单缓存（Map<word, response>）避免同一词重复请求。

### Decision: 下一词跳转策略 —— 正确时自动，错误/超时手动
根据正确/错误/超时三种状态采取不同的跳转策略：
- **正确**：匹配后输入框变绿，0.8 秒后自动跳转下一词；用户按 Enter 可提前跳转
- **错误/超时**：界面冻结，显示正确答案（红色），用户必须手动点击"下一词"按钮

**原因**: 正确时快速推进保持节奏感，同时 Enter 提供加速选项；错误时强制用户停下来审视正确答案，加深记忆。

### Decision: CSS 变量驱动主题
使用 CSS 自定义属性定义颜色、字体、间距等设计 token。
**原因**: 便于后续主题切换（暗色模式等），且易于维护。

## Data Architecture

### 本地词库 (data/words.json)
```json
[
  {
    "word": "abandon",
    "meaning_zh": "v. 放弃，遗弃",
    "phonetic": "/əˈbændən/",
    "example": "They had to abandon the project."
  }
]
```

### localStorage 结构
```json
{
  "spellingTrainer": {
    "wrongWords": [
      {
        "word": "abandon",
        "meaning_zh": "v. 放弃，遗弃",
        "phonetic": "/əˈbændən/",
        "example": "They had to abandon the project.",
        "wrongAt": "2026-06-24T10:30:00.000Z"
      }
    ]
  }
}
```

### API 响应缓存 (运行时 Map)
```typescript
Map<string, { audioUrl: string; phonetics: string; example: string }>
```

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| dictionaryapi.dev 可能被墙或限流 | 核心功能不依赖 API；添加请求超时（3s）和重试逻辑 |
| localStorage 可能被用户清除 | 提供导出功能让用户可备份；首次使用提示数据存储位置 |
| 500 词词库 JSON 文件体积（约 50KB） | Gzip 后极小，不影响加载速度 |
| 移动端虚拟键盘可能遮挡输入框 | 输入框固定在视口可见区域，使用 `visualViewport` API 适配 |
| 浏览器 TTS 与真人发音质量差距 | 使用 dictionaryapi.dev 的真人录音 MP3，TTS 只作为兜底 |
